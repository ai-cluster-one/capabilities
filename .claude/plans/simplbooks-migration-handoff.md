We are migrating this machine from the OLD agent-capabilities model to the v2 manager model, and migrating THIS project's (simplbooks) capability config. Be careful and verify each step; do not delete anything until its replacement is confirmed.

BACKGROUND
- Old model: capability scripts symlinked from ~/bin into the source repo at ~/dev/capabilities; per-project config as markdown under .capabilities/<name>/ (capability.md, identifiers.md, reference.md, sometimes subfolders).
- v2 model: the `capabilities` manager installs a COPY of each script into ~/.capabilities/<name>/, symlinks ~/.local/bin/<name>, and snapshots its declaration. Per-project config is a JSON envelope: a top-level gate .capabilities/settings.json plus per-capability .capabilities/<name>/{connections.json, identifiers.json, settings.json} and references under .capabilities/<name>/reference/*.md. The session injection (.claude/rules/CAPABILITIES.md) is generated by `capabilities context --claude`, wired by `capabilities init`.

CRITICAL — why this is not a rename:
- v2 context generation reads ONLY connections.json / identifiers.json (JSON) and reference .md files whose YAML front-matter contains BOTH a `name:` and a `description:` field. Old markdown (description-only front-matter, or none) is SILENTLY IGNORED — install/enable will not error, but that project knowledge drops out of the agent's context.
- Reference .md files live in .capabilities/<name>/reference/ — a dedicated subfolder, the single home for references, kept apart from the JSON config files. The context build and `<name> refs` glob reference/*.md and read FRONT-MATTER ONLY. A loose .md beside the JSON files, or at the envelope top level, is NOT read.
- identifiers.json holds VALUES ONLY (ids, codes, labels) in the envelope shape `{ "<label>": { "value": <any>, "note": "<text>" } }` — NO prose, NO method, NO narrative (DOCTRINE rule 4). Any "how it works / how to book it" prose is NOT an identifier; it is a reference.

AUTHORITATIVE SHAPES: read /Users/kz/dev/capabilities/TEMPLATE.md (the slots + "Settings vs identifiers vs reference") and /Users/kz/dev/capabilities/SHEBANG.md (the "Connections", "project envelope", and "ids verbs" sections).
