# Railway (railwayc) — reference

> **Purpose.** The standing home for *project-specific* operational context: prose about how **this project** uses Railway, where that's neither a value nor a command. Which environment is which and why, how this project's services relate, any deploy conventions or gotchas specific to this setup.
>
> Structural values (the project the token binds to, service names) live in `identifiers.md`; the harness model — the credential cascade, the `doctor` round-trip, the forward-everything-else rule, the exit codes — lives in `railwayc help`, and the command surface in `railway help` (all project-agnostic, never copied here).
>
> An *executable* deploy or release procedure (which services to redeploy, in what order, on what trigger) is a **project routine** in `.routines/`, not this file; point to it from here, don't embed it.
>
> Ships empty on purpose, so the home is always labeled and no agent has to decide whether it should exist or what goes in it. Populate it only when real context accrues — an empty reference is conformant, not a gap. Replace this note with that content when it arrives.
