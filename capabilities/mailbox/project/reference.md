# Mailbox — reference

> **Purpose.** The standing home for *project-specific* operational context: prose about how **this project** uses its mailbox, where that's neither a value nor the command surface. What a folder means here, a sending or filing convention, a routing quirk in this project's mail, the meaning behind a profile.
>
> Structural values live in their proper homes: profile wiring in `mailbox.json`, the profile roster and secret env keys in `identifiers.md`. How the CLI works — the command surface, discovery rules, the JSON I/O contract, exit codes, the `message_id`-not-`uid` idempotency rule — lives in `mailbox help` (project-agnostic, never copied here).
>
> Ships empty on purpose, so the home is always labeled and no agent has to decide whether it should exist or what goes in it. Populate it only when real context accrues — an empty reference is conformant, not a gap. Replace this note with that content when it arrives.
